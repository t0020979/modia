> 📘 **Для разработчиков приложений** (пользователи фреймворка)

**Версия:** 1.2.0  
**Статус:** ✅ Стабильная

## Основная философия

Лёгкий компонентный фреймворк на JavaScript для управления динамическим UI в legacy-проектах. Совместим с Rails, jQuery, Turbolinks, UJS.

Основная философия: **Расширять без переписывания.**
Modia не требует полной переписки существующего кода — новые компоненты подключаются постепенно, не ломая legacy-функциональность.

- ✅ **Стабильность:** Версии v1.x гарантируют обратную совместимость.
- ✅ **Интеграция:** Работает рядом с jQuery, Turbolinks, UJS и старыми скриптами.
- ✅ **Счастье разработчика:** Понятные ошибки, минимум JS-кода, максимум HTML.

> **💡 Гибридный подход:**
> Modia создан для монолитов, но поддерживает **«Острова» (Islands Architecture)**.
> Вы можете лениво подгружать сложные виджеты (SPA-стиль) внутри обычной страницы, не теряя преимуществ серверного рендеринга.
> См. `AsyncWidgetComponent` (v1.3) и `ErrorBoundaryComponent`.

## 🚀 Быстрый старт

### Подключение

```html
<!-- ✅ jQuery CDN (ОБЯЗАТЕЛЬНО) -->
<script src="https://code.jquery.com/jquery-3.5.1.min.js"></script>

<!-- ✅ Modia (ES6 модуль) -->
<script type="module" src="/modia/index.js"></script>
```

### Базовое использование

```html
<form data-component="validation">
  <div class="form-group">
    <input type="email" name="email" required>
    
    <!-- Шаблон ошибки (Уровень 2 — рекомендуемый) -->
    <div class="error-template">
      <span class="text-danger" data-rule="required">Обязательное поле</span>
    </div>
  </div>
  
  <button type="submit">Отправить</button>
</form>
```

## 🔄 Иерархия сообщений об ошибках (5 уровней)

| Уровень | Источник | Пример | Когда использовать |
| --- | --- | --- | --- |
| 1 | Inline-атрибут | `data-error-text-required="Заполните"` | Точечное переопределение |
| 2 | Шаблон в контейнере | `.error-template [data-rule="required"]` | **Рекомендуемый** |
| 3 | Отдельные шаблоны | `#error_span` | Legacy-код |
| 4 | Сообщение из правила | `defaultMessage` в конфиге | Fallback |
| 5 | Fallback валидатора | `"Field error"` | Критическая ошибка |

## 🚧 Запланировано (v1.3 — v2.0)

| **ErrorBoundaryComponent** | Изоляция ошибок в partials | ⏳ Планируется |
| **DebugPanel Component** | UI панель, диагностика страницы, экспорт логов | ⏳ Планируется |
| **Turbolinks поддержка** | Полная интеграция с `turbolinks:load` | ⏳ Планируется |
| **ENV блокировка логов** | `window.MODIA_LOG_ENABLED`, `window.MODIA_LOG_LEVEL` | ⏳ Планируется |

### v1.4 — Расширение валидации (Q2 2026)

| Модуль | Назначение | Статус |
| --- | --- | --- |
| **Domain Layer** | Чистые бизнес-объекты (без DOM) | ⏳ Планируется |
| **DataTransportLayer** | Абстракция над AJAX (чтение и запись) | ⏳ Планируется |
| **StateReducer** | Управление валидными переходами состояния | ⏳ Планируется |
| **Render Endpoint** | Единый endpoint для рендеринга UI-блоков | ⏳ Планируется |

### v1.5+ — Новые компоненты UI (Q3 2026+)

| Компонент | Назначение | Статус |
| --- | --- | --- |
| **ModalComponent** | Открытие/закрытие, сканирование внутреннего DOM | ⏳ Планируется |
| **DynamicSelectComponent** | Асинхронная подгрузка опций в `<select>` | ⏳ Планируется |
| **DependentFieldComponent** | Блокировка/очистка полей при изменении другого | ⏳ Планируется |
| **TabGroupComponent** | Переключение вкладок с поддержкой `data-component` | ⏳ Планируется |
| **Schema-Driven UI** | Генерация форм из декларативной схемы (JSON) | ⏳ v2.0 |

## 🐛 Debug-режимы

### Активация через URL
```
https://example.com/form?modia-log
https://example.com/form?modia-debug
```

### Активация через data-атрибут
```html
<body data-modia-log>
```

### Логирование в коде
```javascript
Modia.log('Сообщение', 'info', 'Context');
Modia.info('Информация', 'Context');
Modia.warn('Предупреждение', 'Context');
Modia.error('Ошибка', 'Context');
Modia.success('Успех', 'Context');
```

> ⚠️ **Важно:** Логи выводятся только при активном режиме отладки.
